Perguntas

  1. Na tag 'input' com atributo type="text", para que serve o atributo "placeholder"?
  2. Na tag 'input' com atributo type="radio" ou type="checkbox", para que serve o atributo "checked"?
  3. Na tag 'input' para que serve o atributo "value"?
  4. Que atributo deve ser utilizado para agrupar vários botões de rádio em um formulário de modo que apenas um deles fique marcado de cada vez?
  5. Para que serve a tag 'label'? Para que serve o atributo "for" nessa tag?
  6. Na tag 'option' para que serve o atributo "selected"?
  7. Nas tags de formulário para que serve o atributo "required"?
  8. Para que serve a tag 'textarea'?
  9. Na tag 'form' para que serve o atributo "action"?
  10. Na tag 'form' para que serve o atributo "method"?
  11. Para que o valor digitado de uma senha não apareça na barra de endereços do navegador, qual deve ser o valor do atributo "method" na tag 'form'?
  12. Qual valor deve ser informado no atributo "method" na tag 'form' em um formulário que será usado para fazer upload de arquivos?
  13. O que as tags 'fieldset' e 'legend' produzem?

Respostas

  1. O atributo "placeholder" é utilizado para fornecer um texto de exemplo dentro do campo de entrada. Esse texto desaparece quando o usuário começa a digitar.
  2. O atributo "checked" serve para inicialmente selecionar um botão de rádio ou caixa de seleção por padrão.
  3. O atributo "value" define o 'valor' que será enviado quando o formulário for submetido.
  4. Você deve utilizar o atributo "name" com o mesmo valor para cada botão de rádio dentro do grupo.
  5. A tag "label" é usada para associar um rótulo descritivo a um elemento de formulário. O atributo "for" especifica qual elemento de formulário está sendo rotulado.
  6. O atributo "selected" é utilizado para indicar que uma opção está selecionada por padrão em um elemento "select".
  7. O atributo "required" é usado para indicar que um campo deve ser preenchido antes que o formulário possa ser enviado (submit).
  8. A tag "textarea" é usada para criar uma área de texto multi-linhas onde os usuários podem digitar ou colar texto.
  9. O atributo "action" especifica para onde os dados do formulário serão enviados quando o formulário for 'submitido'.
  10. O atributo "method" especifica o método HTTP a ser usado ao enviar os dados do formulário.
  11. A segurança da senha é garantida pelo uso do atributo 'type="password"' na tag "input". O valor do atributo "method" não tem relação com essa questão.
  12. O valor do atributo "method" deve ser "POST", pois os dados do formulário, incluindo os arquivos enviados, são codificados e incluídos no corpo da requisição HTTP quando o método POST é utilizado.
    • GET: Este método envia os dados do formulário como parte da URL. É adequado para formulários simples que não contêm informações sensíveis e quando você deseja que os dados do formulário sejam visíveis na barra de endereços do navegador. No entanto, existe um limite de tamanho para os dados que podem ser enviados usando o método GET.
    • POST: Este método envia os dados do formulário no corpo da requisição HTTP. Ele é adequado para formulários que contêm informações sensíveis ou quando você precisa enviar grandes quantidades de dados, como ao fazer upload de arquivos. Os dados enviados usando o método POST não são visíveis na barra de endereços do navegador.
  13. As tags "fieldset" e "legend" são usadas para agrupar elementos de formulário relacionados em uma área visualmente distinta. "fieldset" cria uma caixa ao redor dos elementos, enquanto "legend" especifica uma legenda ou título para o grupo de elementos dentro do "fieldset".
